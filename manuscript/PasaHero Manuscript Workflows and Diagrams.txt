Program Workflow
This section describes the program workflow of the proposed PasaHERO system. It illustrates the sequence of user interactions and automated system processes that work together to provide intelligent public transportation navigation in Cebu City. The workflow demonstrates how user inputs, real-time location data, jeepney route information, landmark records, and fare data are processed by the AI-Based Route Recommendation Engine to generate multimodal route recommendations, interactive route visualizations, fare estimates, and navigation assistance. It also illustrates how the application continuously monitors the commuter's location to provide destination notifications ("Para" alerts) upon approaching the designated drop-off point. The PasaHERO program workflow consists of four interconnected workflows: the Overall Program Workflow, User Workflow, System Workflow, and Admin Workflow, each representing the major operational processes that support the functionality and maintenance of the proposed system. 
Detailed Workflow
Illustrates the overall operation of the PasaHERO application, beginning with user authentication and ending with trip completion. After logging into the system, commuters provide their current location through GPS or manual input and enter their intended destination. The system retrieves the necessary route, fare, and landmark information before the AI-Based Route Recommendation Engine evaluates multiple contextual travel factors, including walking distance, estimated fare, route availability, number of transfers, travel distance, and nearby landmarks. Each feasible route is assigned a recommendation score based on predefined recommendation criteria. The available routes are then ranked according to their recommendation scores, and the highest-ranked multimodal route is selected as the recommended route for the commuter.
The selected route is presented through an interactive map displaying jeepney routes, walking paths, boarding and drop-off points, nearby landmarks, estimated fares, and estimated travel time. During navigation, the application continuously tracks the commuter's real-time location, monitors route progress, and evaluates geofence boundaries to trigger a Destination Notification ("Para" Alert) when the commuter is approaching the designated drop-off location. Once the destination is reached, the system generates a trip summary containing the recommended route taken, estimated fare, travel distance, and travel time, saves the trip history in the database, and allows commuters to optionally provide feedback regarding their travel experience before completing the trip.

User Workflow
The User Workflow illustrates how commuters interact with the PasaHERO application throughout their journey. The workflow begins when users access the application and either log in to an existing account or register a new one. After authentication, the system verifies the user's credentials before directing them to the user dashboard. Commuters then set their trip details by providing their current location through GPS auto-detection or manual input and entering their intended destination. The system subsequently generates and displays an AI-powered recommended route that includes the appropriate jeepney route and signboard, first-mile and last-mile walking paths, estimated fare and travel time, boarding and drop-off points, and nearby landmarks. Users then start navigation and receive real-time route guidance, live location updates, and walking directions throughout the trip. As the commuter approaches the designated drop-off point, the application sends a geofenced "PARA" notification to remind the user to prepare for alighting. Once the destination is reached, the system presents a trip summary containing the route taken, estimated fare, distance traveled, and travel time. Users may optionally provide feedback by rating their travel experience before ending their session by logging out of the application. 

System Workflow
The System Workflow describes the internal operations performed by the proposed PasaHERO system after receiving a commuter's travel request. The process begins when the user provides the current location and destination, after which the system validates the input and retrieves the necessary data from the MySQL database, including jeepney routes, route codes and signboards, landmarks and stops, and LTFRB-approved fare rates. The AI-Based Route Recommendation Engine then evaluates all feasible routes by considering multiple contextual travel factors such as walking distance, estimated fare, route availability and frequency, number of transfers, travel distance, and nearby landmarks and safety. Each route is assigned a recommendation score and ranked according to its overall suitability before the highest-ranked route is selected and presented through an interactive map showing the jeepney route, boarding and drop-off points, walking directions, estimated fare, estimated travel time, and nearby landmarks. During the trip, the system continuously tracks the commuter's location using GPS to provide real-time navigation and automatically sends a geofenced "PARA" notification when the commuter approaches the designated drop-off point. Upon reaching the destination, the system generates a trip summary containing the route taken, fare, travel time, and trip date, allows the commuter to submit feedback, and stores the completed trip in the MySQL database for future reference. 

Admin Workflow
The Admin Workflow illustrates the administrative processes responsible for maintaining the accuracy, reliability, and efficiency of the proposed PasaHERO system. The workflow begins when the administrator securely logs into the admin dashboard using authorized credentials, where system statistics and recent activities are displayed. From the dashboard, the administrator manages jeepney routes, route codes and signboards, LTFRB fare rates, landmarks, jeepney stops, and user accounts to ensure that all navigation data remains accurate and up to date. The administrator also monitors reports and analytics, reviews trip records and statistics, manages announcements and notifications, configures system settings and user permissions, and performs database backup, restoration, maintenance, and integrity checks to preserve data reliability. All administrative updates are securely stored in the MySQL database, allowing the latest information to be reflected throughout the application and ensuring commuters receive accurate route recommendations, fare information, and real-time navigation services before the administrator securely logs out of the system.

The First-Mile Navigation (User) screens guide commuters from their current location to the designated jeepney boarding point using live map navigation. The Transportation Mode (User) screens provide real-time navigation while commuters traveling on the recommended jeepney route. The User Profile and Saved Places (User) screens allow commuters to manage their personal account information and frequently visited destinations. The Trip History (User) screens allow commuters to review previously completed trips recorded by the PasaHERO mobile web application.
